Dalam konteks plugin GPT, ada dua file penting yang harus disiapkan dan ditempatkan di server Anda:

ai-plugin.json – Ini adalah manifest plugin. File ini memberi tahu ChatGPT nama plugin, deskripsi singkat, jenis autentikasi, logo, kontak, dan — yang paling penting — URL untuk file spesifikasi OpenAPI Anda. Manifest ini harus disimpan di https://[domain-anda]/.well-known/ai-plugin.json agar ChatGPT bisa menemukannya.

openapi.yaml – Ini adalah spesifikasi API Anda. File ini menjelaskan semua endpoint yang tersedia (/api/sol/complete, /health, dsb.), metode HTTP yang digunakan, parameter, dan contoh respons. ChatGPT menggunakan file ini untuk memahami cara memanggil API Anda secara otomatis.

Ketika saya menulis “pastikan file ai-plugin.json dan openapi.yaml mengarah ke domain ini,” maksudnya adalah:

Ganti placeholder domain di dalam kedua file. Banyak template plugin yang menggunakan https://your-domain.com atau localhost sebagai contoh. Anda harus menggantinya dengan domain asli tempat server Anda berjalan, misalnya https://guardiansofthegreentoken.com. Contoh:

// di ai-plugin.json
{
  "name_for_human": "Crypto Data Gateway",
  "name_for_model": "crypto_data_gateway",
  "description_for_human": "Aggregates SOL-USDT data from OKX.",
  "api": {
    "url": "https://guardiansofthegreentoken.com/openapi.yaml"
  },
  ...
}

# di openapi.yaml
servers:
  - url: https://guardiansofthegreentoken.com


Sajikan file tersebut dari domain yang sama. Untuk GPT menemukan plugin Anda, ai-plugin.json harus tersedia di path /.well-known/ai-plugin.json dan openapi.yaml harus bisa diakses di path yang Anda tentukan (mis. /openapi.yaml) di domain itu. Contohnya:

https://guardiansofthegreentoken.com/.well-known/ai-plugin.json

https://guardiansofthegreentoken.com/openapi.yaml

Jika kedua file ini masih menunjuk ke domain dummy, GPT‑5 akan mencoba mengakses alamat yang salah dan gagal memanggil API Anda. Dengan mengganti URL dan memastikan file dapat diakses publik, Anda memberi tahu GPT‑5 di mana harus membaca spesifikasi API Anda dan bagaimana memanggilnya dengan benar.
